# AI Assistant Showcase

AI Assistant демо: backend на FastAPI, RAG по CSV-каталогу в Qdrant, вызовы Groq-совместимой LLM через LiteLLM, Telegram-бот и vanilla JS виджет.

В текущем демо-примере используется каталог юридической компании: услуги по договорам, персональным данным, судебным спорам, корпоративным вопросам и тарифы юридического сопровождения бизнеса.

## Быстрый Старт

1. Создать env-файлы: `make env-init`.
2. Указать `LLM_API_KEY` в `.env.dev`. Укажи `TELEGRAM_BOT_TOKEN`, только если хочешь запускать Telegram-бота.
3. Запусти сервисы: `make run-dev`.
4. Примени миграции: `make migrate-head`.
5. Импортируй каталог юридической компании: `make import-catalog-csv`.
6. Прогрей embedding-модель в работающем backend-процессе: `make warmup-dev`.
7. Проверь chat API:

```bash
curl -X POST http://localhost:8000/api/chat/message \
  -H 'Content-Type: application/json' \
  -d '{"tenant_slug":"demo","message":"У меня небольшой интернет-магазин. Нужны оферта, политика конфиденциальности и проверка сайта на юридические риски. Что посоветуете?"}'
```

8. Открой `widget/index.html` в браузере, чтобы проверить web-виджет.

## Пример Диалога

Для проверки демо можно задать вопросы из юридического сценария:

```text
У меня небольшой интернет-магазин. Нужны оферта, политика конфиденциальности и проверка сайта на юридические риски. Что посоветуете?
```

```text
Контрагент не оплатил услуги по договору. Что лучше сделать сначала и какие документы подготовить?
```

```text
У нас регулярно появляются договоры и юридические вопросы. Какой тариф выбрать: Start, Business или Premium?
```

## Чистый Импорт

Если нужно загрузить каталог с нуля для tenant-а `demo`, можно очистить PostgreSQL-данные tenant-а и Qdrant collection:

```bash
make clear-postgres
make clear-qdrant
make import-catalog-csv
```

Для другого tenant-а передай `TENANT_SLUG` в команды очистки и `--tenant-slug` в команду импорта.

## LLM Модель

LiteLLM ожидает модель с префиксом провайдера. Для Groq указывай модель в таком формате:

```env
LLM_MODEL=groq/llama-3.3-70b-versatile
```

Без префикса `groq/` LiteLLM не сможет определить провайдера.

## Telegram

Запусти бота после настройки `TELEGRAM_BOT_TOKEN` и импорта каталога:

```bash
make run-telegram-dev
```

## Проверки

Используй `make check`, чтобы запустить lint, проверку форматирования и тесты внутри Docker.
